== Overview

DwarFS v0.16.0 has **1,613 total tests** organized across multiple test suites. Tests are **conditionally compiled** based on available metadata formats, resulting in different test counts per build configuration.

**Key Principle**: "100% of applicable tests pass" - not all tests apply to all builds.

== Format-Specific Tests

=== Thrift-Specific Tests (13 tests)

These tests skip when `DWARFS_WITH_THRIFT=OFF`:

==== 1. Direct Thrift Type Usage (4 tests)

**File**: `test/global_metadata_test.cpp`
- **Why**: Uses `thrift::metadata::metadata` types directly
- **Tests**: Frozen2 consistency checks, schema validation
- **Skip Reason**: Thrift types don't exist without Thrift

**File**: `test/metadata_test.cpp`
- **Why**: Uses `metadata_freezer` (Thrift-specific)
- **Tests**: Metadata rebuild with Thrift types
- **Skip Reason**: Thrift serialization unavailable

**File**: `test/metadata/format_conversion_test.cpp`
- **Why**: Tests Thrift format conversion
- **Tests**: Conversion between Thrift and domain model
- **Skip Reason**: Thrift conversion unavailable

**File**: `test/block_merger_test.cpp`
- **Why**: Uses `folly::Synchronized` (Folly dependency)
- **Tests**: Concurrent block merging
- **Skip Reason**: Folly unavailable without Thrift

==== 2. Pre-Built Thrift Images (1 test)

**File**: `test/filesystem_test.cpp`
- **Why**: Uses pre-built test images in Thrift format:
  - `winlink.dwarfs`
  - `unixlink.dwarfs`
  - `compat/compat-v0.2.0.dwarfs`
  - `compat/compat-v0.9.10.dwarfs`
  - `future-features.dwarfs`
- **Tests**: Reading old DwarFS versions
- **Skip Reason**: Cannot read Thrift images without Thrift support

==== 3. Both-Format Tests (8 tests)

**File**: `test/backend_compatibility_test.cpp`
- **Why**: Requires BOTH formats (`#if FLATBUFFERS && THRIFT`)
- **Tests**: Cross-format compatibility
- **Skip Reason**: Needs both formats to test compatibility

=== FlatBuffers-Specific Tests (17 tests)

These tests skip when `DWARFS_WITH_FLATBUFFERS=OFF`:

==== 1. FlatBuffers Magic Bytes (1 test)

**File**: `test/metadata_factory_test.cpp`
- **Tests**: FlatBuffers magic byte detection ("DFBF")
- **Skip Reason**: FlatBuffers format unavailable

==== 2. FlatBuffers Serialization (8 tests)

**File**: `test/metadata/serialization_test.cpp`
- **Tests**: FlatBuffers serialize/deserialize round-trips
- **Skip Reason**: FlatBuffers serializer unavailable

==== 3. FlatBuffers String Compression (3 tests)

**File**: `test/metadata/compact_names_test.cpp`
- **Tests**: FSST string table compression (FlatBuffers-specific)
- **Skip Reason**: FlatBuffers string compression unavailable

==== 4. Asymmetric Tests (5 tests)

**Reason**: FlatBuffers has some additional tests:
- Read-write symmetry tests
- Size verification tests
- Magic byte tests not applicable to Thrift

== Test Asymmetry Explained

=== Why 17 FlatBuffers Tests vs 16 Thrift Tests?

The asymmetry is **intentional and correct**:

**FlatBuffers (17 tests)**:
- 8 serialization tests
- 3 string compression tests
- 1 magic byte test
- 5 other format-specific tests

**Thrift (16 tests)**:
- 8 serialization tests
- 8 other format-specific tests

**Extra FlatBuffers tests**:
1. **Magic byte detection** - Thrift has no magic bytes (falls back)
2. **FSST string compression** - Thrift uses different string storage
3. **Read-write symmetry** - FlatBuffers needs explicit testing

This is **not a bug** - it reflects genuine differences in format capabilities.

== Running Tests

=== Run All Applicable Tests

[source,bash]
----
# FlatBuffers-only build
cmake -B build-fb -DDWARFS_WITH_FLATBUFFERS=ON \
  -DDWARFS_WITH_THRIFT=OFF -DWITH_TESTS=ON
ninja -C build-fb
ctest --test-dir build-fb --output-on-failure

# Expected: 1,600 pass, 13 skip
----

=== Run Specific Test Suites

[source,bash]
----
# Unit tests only
ctest --test-dir build -R dwarfs_unit_tests

# Metadata tests only
ctest --test-dir build -R metadata

# Format-specific tests
ctest --test-dir build -R serialization

# Tool integration tests
ctest --test-dir build -R tool_main
----

=== Verify Test Matrix

[source,bash]
----
#!/bin/bash
# Test all 3 configurations

configs=(
  "fb-only:-DDWARFS_WITH_FLATBUFFERS=ON -DDWARFS_WITH_THRIFT=OFF:1600:13"
  "thrift-only:-DDWARFS_WITH_FLATBUFFERS=OFF -DDWARFS_WITH_THRIFT=ON:1596:17"
  "both:-DDWARFS_WITH_FLATBUFFERS=ON -DDWARFS_WITH_THRIFT=ON:1613:0"
)

for config in "${configs[@]}"; do
  IFS=':' read -r name cmake_args expected_pass expected_skip <<< "$config"
  
  echo "Testing $name configuration..."
  cmake -B build-$name -GNinja $cmake_args -DWITH_TESTS=ON
  ninja -C build-$name
  
  ctest --test-dir build-$name --output-on-failure
  
  # Verify counts
  passed=$(ctest --test-dir build-$name --quiet | grep -c "Passed")
  skipped=$(ctest --test-dir build-$name --quiet | grep -c "Skipped" || echo 0)
  
  if [ "$passed" -eq "$expected_pass" ] && [ "$skipped" -eq "$expected_skip" ]; then
    echo "✅ $name: PASS ($passed pass, $skipped skip)"
  else
    echo "❌ $name: FAIL (expected $expected_pass/$expected_skip, got $passed/$skipped)"
  fi
done
----

== Test Failures vs Skips

=== Expected Skips ✅

**Characteristics**:
- Test name includes format name (e.g., `ThriftSpecific*`)
- Guarded by `#ifdef DWARFS_HAVE_THRIFT`
- Console shows: `[  SKIPPED  ]`
- Exit code: 0 (success)

**Example**:
```
[  SKIPPED  ] GlobalMetadataTest.ThriftFrozenConsistency
[  SKIPPED  ] MetadataSerializationTest.ThriftRoundTrip
```

=== Actual Failures ❌

**Characteristics**:
- Test runs but fails assertion
- Console shows: `[  FAILED  ]`
- Exit code: non-zero
- Indicates actual bug

**Example**:
```
[  FAILED  ] FileSystemTest.ReadFile
Expected: data == expected
  Actual: data != expected
```

**Action**: Report as bug if outside expected skip list.

== Writing New Tests

=== Format-Agnostic Tests (Preferred)

[source,cpp]
----
TEST(MyTest, WorksWithBothFormats) {
  // Test using abstract interfaces, not format-specific types
  auto provider = make_metadata_provider(data);
  auto chunk = provider->get_chunk(0);
  EXPECT_EQ(chunk.size(), expected_size);
}
----

=== Format-Specific Tests (When Necessary)

[source,cpp]
----
#ifdef DWARFS_HAVE_FLATBUFFERS
TEST(FlatBuffersTest, MagicByteDetection) {
  std::vector<uint8_t> data = create_flatbuffers_data();
  EXPECT_TRUE(has_flatbuffers_magic(data));
}
#endif

#ifdef DWARFS_HAVE_THRIFT
TEST(ThriftTest, FrozenConsistency) {
  thrift::metadata::metadata meta = create_thrift_metadata();
  EXPECT_TRUE(validate_frozen_layout(meta));
}
#endif
----

== Troubleshooting

=== More Tests Skipping Than Expected

**Possible Causes**:
1. Build configuration mismatch
2. Missing dependencies
3. Test compilation errors

**Solution**:
[source,bash]
----
# Check CMake configuration
cmake -B build -LAH | grep DWARFS_WITH

# Rebuild tests
ninja -C build clean
ninja -C build
----

=== Tests Failing That Should Skip

**Possible Cause**: Conditional guards missing or incorrect

**Solution**: Check test source for proper `#ifdef` guards

=== Both-Format Build Has Skips

**Problem**: Should be 0 skips with both formats enabled

**Solution**: Verify both `DWARFS_WITH_FLATBUFFERS=ON` and `DWARFS_WITH_THRIFT=ON`
