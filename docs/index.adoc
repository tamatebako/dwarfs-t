== Welcome to DwarFS

DwarFS is a fast, high-compression read-only file system designed for extreme compression ratios while maintaining fast random access performance. It functions as a mountable archive that can be browsed instantly without full extraction.

**Version**: 0.16.0 +
**Release Date**: 2025-12-15 (target)

=== Key Features

* **Extreme compression**: 100:1+ ratios on redundant data
* **Fast random access**: Memory-mapped, zero-copy access
* **Multi-format metadata**: FlatBuffers (modern default) and Thrift Frozen2 (legacy)
* **Cross-platform**: Linux, FreeBSD, macOS, Windows
* **Multi-architecture**: x86_64, aarch64, arm, ppc64, riscv64, s390x, loongarch64

== What's New in v0.16.0

=== Multi-Format Metadata Serialization

v0.16.0 introduces a flexible multi-format metadata serialization architecture:

* **FlatBuffers** as the modern default format
* **Thrift Frozen2** maintained for legacy compatibility
* **Strategy Pattern** implementation for clean format abstraction
* **Build-time format selection** via CMake options

See link:_guides/multi-format-architecture.html[Multi-Format Architecture Guide] for details.

=== Critical Fixes

* Fixed FlatBuffers buffer overflow in verification (affected large filesystem images)
* Increased verifier limits: max_depth 64→256, max_tables 1M→10M

== Getting Started

=== Quick Start

[source,bash]
----
# Install dependencies (Ubuntu/Debian)
sudo apt-get install -y cmake ninja-build libboost-all-dev \
  libssl-dev libzstd-dev libxxhash-dev libarchive-dev

# Clone repository
git clone --recurse-submodules https://github.com/tamatebako/dwarfs
cd dwarfs

# Build with FlatBuffers (recommended)
cmake -B build -GNinja -DDWARFS_WITH_FLATBUFFERS=ON
ninja -C build

# Create your first filesystem
./build/mkdwarfs -i /path/to/source -o myfs.dwarfs

# Mount and browse
./build/dwarfs myfs.dwarfs /mnt/point
----
