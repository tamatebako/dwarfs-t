fix(reader): add backend adapter to resolve architecture mismatch

Session 33: Reader layer architecture fix for metadata serialization.

Problem:
--------
common_metadata_operations::get_chunks() was constructing chunk_range
directly with domain model at line 1134, but different build configs
expect different types:

- FlatBuffers-only: chunk_range = domain_chunk_range_impl (✅ domain-native)
- Thrift-only: chunk_range = thrift_backend::chunk_range (❌ expects frozen)
- Both-formats: chunk_range = chunk_range_wrapper (❌ needs interface)

This caused Thrift-only and both-formats builds to fail.

Solution:
---------
Created backend_adapter to bridge domain model → backend-specific types:

1. backend_adapter::make_chunk_range() handles all three build configs:
   - FlatBuffers-only: Direct construction
   - Both-formats: Wraps domain impl in interface
   - Thrift-only: TODO (throws runtime error for now)

2. Updated common_metadata_operations.cpp to use adapter instead of
   direct construction.
